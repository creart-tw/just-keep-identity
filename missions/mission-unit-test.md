# Mission: Just Keep Identity (jki) Unit Testing & Coverage

## 1. Project Context
**Just Keep Identity (jki)** is a high-speed MFA/TOTP vault built in Rust. It emphasizes:
- **Speed**: Cold start < 3ms.
- **Security**: Split-data architecture (Metadata vs. Encrypted Secrets).
- **Ergonomics**: One-handed operation (j-k-i).

### Current Architecture (Rust Workspace):
- `crates/jki-core`: Core logic (Search, Path handling, Age-based Crypto, Auth logic).
- `crates/jki`: The high-speed CLI executor (Search & OTP generation).
- `crates/jkim`: The management hub (Import, Git sync, Status).
- `data/private/`: Local storage for testing (Metadata JSON + Age-encrypted Secrets).

## 2. Objective
Implement a comprehensive unit test suite across the workspace to ensure reliability and achieve high code coverage.

## 3. Key Logic to Test
### jki-core
- **Fuzzy Match**: Test `fuzzy_match` with various patterns (e.g., `gg` matches `google`).
- **Search Engine**: Test `search_accounts` intersection logic.
- **Crypto**: Test `encrypt_with_master_key` and `decrypt_with_master_key` round-trip.
- **Paths**: Test `JkiPath` resolution and permission checks (mocking OS if needed).
- **Import**: Test `parse_otpauth_uri` with various WinAuth URI formats.

### jki / jkim
- Command-line argument parsing (Clap integration).
- Data integration logic (matching metadata IDs with secrets).

## 4. Testing Requirements
- **In-source Unit Tests**: Place them at the bottom of `.rs` files.
- **Integration Tests**: Use `tests/` directory for end-to-end flows.
- **Edge Cases**: Empty vaults, missing secrets, wrong master keys, duplicate entries.
- **Mocking**: Be prepared to mock TTY interactions (`crossterm`, `rpassword`) or focus tests on pure logic wrappers.
